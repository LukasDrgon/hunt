hunt

by Jeremias Menichelli

Library to detect when DOM elements become visible and disappear on scroll.

You can download the code from the github repository or install the npm package as huntjs.

Support: Chrome, Firefox, Safari, Edge and IE9+

Use

Once you've included the script tag or require the module you need to simply pass an element or a list of them and an object to configure the actions and behaviors.

hunt(document.getElementsByClassName('element'), {
    enter: function() {
        this.classList.add('visible');
    },
    out: function() {
        this.classList.remove('visible');
    }
});

You don't need to pass both in and out, you can pass either one of them or both, of course. You might have also noticed that inside those methods you can reference the element that has become visible using the reserved word this to apply any modification to it.

By default hunt will stop "hunting" or watching for the element once the out method has been executed to improve performance, but if you need these methods to be called every time the element appears and disappears from the viewport you can pass a persist option as true, but beware you can affect scrolling performance.

hunt(document.getElementsByClassName('element'), {
    enter: function() {
        this.classList.add('visible');
    },
    out: function() {
        this.classList.remove('visible');
    },
    persist: true
});

In case you need actions to be executed under the hood, you can use the offset option.

hunt(document.querySelector('.action--element'), {
    enter: function() {
        this.classList.add('visible');
    },
    persist: false,
    offset: 150
});

Toggle a class and animate using CSS

Pretty popular behaviour seen in a lot of sites nowadays.

hunt(document.getElementsByClassName('sample--one')[0], {
    enter: function() {
        this.classList.add('visible');
    },
    out: function() {
        this.classList.remove('visible');
    },
    persist: true
});

Use this library to toggle a class and let CSS rule the animation to save performance and create great effects.

Easily lazy load images

Put a data-src attribute in the image and let hunt do the job.

hunt(document.getElementsByClassName('sample--two-image'), {
    enter: function() {
        this.src = this.dataset.src;
    }
});

Do things only once

By default the persist option is false, so this will only be executed once.

hunt(document.getElementsByClassName('sample--three')[0], {
    enter: function() {
        this.classList.add('visible');
    }
});

User will not see this animation more than once because we are telling hunt that hunting should not persist.
